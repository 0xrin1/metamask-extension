## Token Tiers

We follow a 3 tiered system for color design tokens and css variables.

### **Brand colors** (tier 1)

These colors **SHOULD NOT** be used in your styles directly they are used as a reference for the [Light Theme Colors](/story/design-tokens-color-light-theme-colors--page). Brand colors should just keep track of every color used in our app.

### **Light & Dark Theme Colors** (tier 2)

These colors are color agnostic, semantically neutral and theme compatible design tokens that you can use in your code and styles. Please refer to the description of each token for it's intended purpose. [See Light Theme Colors](/story/design-tokens-color-light-theme-colors--page)

### **Component colors** (tier 3)

Another level of abstraction is component colors that you can use at a component specific level

## Takeaways

- Do not use static HEX values in your code. Use the [Light Theme Colors](/story/design-tokens-color-light-theme-colors--page). If one does not exist for your use case [create an issue](https://github.com/MetaMask/metamask-extension/issues/new) and tag it with a `design-system` label.
- Make sure the design token you are using is for it's intended purpose. Please refer to the description of each token.

### ‚ùå Don't do this

Don't use static hex values or brand color tokens in your code.

```css
/**
* Don't do this 
* Static hex values create inconsistency and will break UI when using dark mode 
**/
.card {
  background-color: #ffffff;
  color: #24272a;
}

/**
* Don't do this
* Not theme compatible and will break UI when using dark theme 
**/
.card {
  background-color: var(--brand-colors-white-white000);
  color: var(--brand-colors-grey-grey800);
}
```

### ‚úÖ Do this

Do use component tiered and [Light Theme Colors](/story/design-tokens-color-light-theme-colors--page) in your styles and code

```css
.card {
  --color-card-background: var(--color-background-default);
  --color-card-text: var(--color-text-default);

  background-color: var(--color-card-background);
  color: var(--color-card-text);
}
```
